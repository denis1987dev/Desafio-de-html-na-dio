Introdução do HTML 5

O HTML 5 é a versão mais nova já criada.

( voltar )

  • Fundamentos: Hyper Text Markup Language / linguagem de marcação de hipertexto;
    Linguagem de marcação.
  • História Como surgiu: Dentro do laboratório da Cern.
  • Criador: Tim Berners-lee e Roberto Cailliau;
    Texto ao qual se agregam outros conjuntos de informação, os famosos links; A ideia do Tim era facilitar a troca, agrupamento e atualização de informações de documentos do mundo todo, e é assim que surgiu uma organização que é a W3C (World Wide Web Consortium);
  • Padronizaram o HTML: HTML 2.0 / 1995, HTML 3.2 / 1997, HTML 4.01 / 1999, HTML 5 /2008
  • A importância de começar pelo HTML Básico mas importante; Html seria o corpo; CSS seria a roupinha; Fácil de começar; Feedback mais visual

Criação da primeira página HTML

( voltar )

  • A organização é essencial para desenvolvedor deste do inicio.
  • Os arquivos têm ícones diferentes para você saber qual é o arquivo que esta sendo utilizado.
    • ( <> HTML ) ( # CSS )
  • O Index é padrão em todos os servidores, ele fica na raiz do arquivo.
  • O servidor é onde nós vamos carrega os arquivos que estão sendo criado.
  • A estrutura básica do HTML:
    ( < html> ): O individuo por completo.
    ( < head> ): A cabeça do individuo.
    ( < / head> ): A cabeça do individuo.
    ( < body> ): O corpo do individuo.
    ( < / body> ): O corpo do individuo.
    ( < / html> ) O individuo por completo.
  • Corpo HTML
    Criar estrutura padrão do HTML, toda a página começa com um comando de HTML que identifica que essa página é HTML e esse comando é o DOCTYPE.
    Em termo de hieraquia o HTML seria o pai, e o HEAD e BODY seriam irmão.
    ( < html>
    < head>
    < / head>
    < body>
    < / body>
    < / html> )
  • Escrever toda a estrutura do HTML usando emmet.
    • 1ª opção posso usa HTML: 5 ou
    2ª opção posso usa ! ponto de exclamação.

Meta tags - cinfiguração informações da sua página

( voltar )

  • Meta tag ou scription: É uma tag que usamos pra descrever uma página da web, e essa descrição vão aparece no mecanismo de busca ex: google, yahoo.
  • Quando estive dentro de uma página da web aperta CTRL + U pra aparece o código fonte da página.
  • Para fazer uma busca dentro do código fonte CTRL + R ele buscar todas as informação.
  • Dentro de uma tag posso passar um atributo é
    • ( < meta charset="UTF-8"> ) charset é atributo e o "UTF-8" o valor, o valor passando para esse atributo.
  • Na tag meta não precisa ter tag de fechamento porque esta simplesmente dando informação da página.
  • A meta tag charset serve pra codificar o formato da nossa página, e é um padrão Unicode que é chamado, como ele pode ser representado quaisquer caracteres de qualquer idioma.
  • Pode se passar mais de dois atributo dentro de uma meta tag.
    • ( < meta name="viewport" ): esse tag vai fornecer pra o navegador informação instrução de como controlar as dimensões e a escala da nossa página.
    ( Content="width=device-width ): define a largura da página que vai se adequar a largura do dispositivo que estamos acessando.
    ( initial-scale=1.0 ): define o nivel de zoom inicial quando a página é carregada pelo navegador.
  • Lorem: é um parágrafo de marcação por exemplo. E o F12 do chrome abre o inspeciona do navegador.
  • A meta tag “description” ela descrever oque o site faz.

Titulos e Parágrafos

( voltar )

  • Os títulos são representado pela tag h e um número, o h significa head “cabeçalho” e o numero significa a relevância da tag na página “quanto maior o número menos importante, e quanto menor o número mais importante é” ( a tag h1 vai até o h6) assim posso criar uma hierarquia entre eles.
  • As tags de “h1 ao h6” vão mudando o tamanho da fonte do número (1 maior até ao 6 menor).
  • Lembrete: Sempre coloque um h1 por página, nunca coloca dois há não ser que eles venham fazer sentido.
  • A tag p é de parágrafo.

Formatação de Texto

( voltar )

  • A tag para deixa o texto em Negrito é o "B" (bold). Mas a tag Strong tem uma semântica melhor. Para uma relevância usa a tag strong.

Comentários

( voltar )

  • Os comentários são trechos de código que não visíveis para o usuário final (cliente), só vai esta visível pra quem esta usando VS Code.
  • Os comentários são diferente das tags ex:
    Isso é um comentário: ( < ! -- isso que usa para fazer um comentário --> )
  • Para escrever um comentário é um pouco diferente das tag .
  • Posso comentar um código pelo comando que é ( ctrl + : ) e ( ctrl + ; ) pra tira só aperta de novo.
  • Comentário posso usa como anotação.
  • Se eu quiser comentar um código aleatório posso usa um comando, mas primeiro precisa selecionar, que é ( ctrl + : dois pontos ).
  • Posso usa o comentário para fazer anotação para nós lembrar que fazer algo a mais, TODO significa a fazer.

Imagens

( voltar )

  • Para criar uma tag no HTML vou criar uma tag “img”.
  • A tag “src” é onde fica a origem do arquivo para carregar. Criar uma pasta para organizar as imagens com o nome de “src” (origem).
  • A tag “alt” (alternativa) serve para fala o que tem dentro daquela imagem quando estiver “quebrada”.
  • Pode colocar uma imagem através do URL de algum site.

Listas

( voltar )

  • Temos dois tipo lista (a ordenada e a não ordenada).
  • A tag “ul” (lista não ordenada) para cada elemento dentro da lista preciso criar uma tag que é “li” (item da lista).
  • Com o “emmet” você pode criar uma lista bem fácil e rápido com “ul>li*4”.
  • Lista ordenada tem a tag “ol” (lista ordenada).

Tabelas

( voltar )

  • Antigamente os site era escritos em tabela.
  • Hoje em dia usamos a tag “table” para criar dados tabulados.
  • Estrutura de uma tabela:
    Th - table header (cabeçalho)
    Tr - table row (linha)
    Td - table data (dados)
  • Com o emmet podemos sua esse atalho:
    ( table>tr*3>td*5 )
    Se o comando não aparecer usa o atalho (ctrl + space)
